Expérience utilisateur

Parcours dans l’application

Cliquez sur une étape pour afficher le détail — du login aux actions sur les fiches employés.

  1. L’utilisateur arrive sur v_login.php ; le contrôleur d’authentification vérifie les identifiants et initialise la session avant toute page sensible.

  2. Le C_dashboard agrège les informations utiles et charge v_dashboard.php pour orienter vers la liste, la recherche ou la saisie.

  3. Consultation, recherche, ajout, modification ou suppression passent par des contrôleurs dédiés qui délèguent aux modèles M_employe et aux vues v_*Employe*.

  4. Les services et employés restent cohérents grâce au schéma SQL et aux classes métier dans metiers/, avec traçabilité des opérations côté MySQL.

Contexte

Vue d’ensemble

Le dépôt pages/Projets/GestionDuPersonnel-MVC-main/S2-MVC contient le point d’entrée index.php, des contrôleurs par cas d’usage, des vues PHP et des modèles qui encapsulent l’accès MySQL. L’objectif est de montrer une chaîne complète : authentification, navigation protégée, puis opérations métier sur les employés et les services.

Les écrans couvrent l’accueil, le tableau de bord, la liste et le détail des employés, l’ajout et la modification, la recherche, ainsi que la gestion des comptes utilisateur selon les scripts fournis. La base est décrite par les fichiers SQL du dossier (schéma employés / services, migrations documentées).

Technique

Architecture MVC (PHP)

Chaque requête HTTP est routée vers un contrôleur qui orchestre les modèles (accès BDD) et choisit la vue à inclure. Les classes métier dans metiers/ portent la logique métier, tandis que modeles/ concentre PDO et requêtes.

MODELE Requêtes SQL, accès employés / services / auth, classes M_*
VUE Fichiers v_*.php : HTML, messages, formulaires
CONTROLEUR C_*.php : actions, validation, redirections

Données & sécurité

Base MySQL et bonnes pratiques

La cohérence des données repose sur le schéma fourni (empsce.sql, variantes et migrations). Côté application, l’usage de requêtes paramétrées dans les modèles limite le risque d’injection SQL.

  • Scripts de migration et guides dans INSTRUCTIONS_MIGRATION.md et DOCUMENTATION_COMPLETE.md
  • Sessions PHP pour l’état de connexion et protection des pages sensibles
  • Séparation stricte des rôles fichiers (vues sans logique SQL lourde)

Développement

Lancer l’application en local

PHP intégré suffit pour servir le dossier S2-MVC une fois la base importée et les paramètres de connexion renseignés (voir README).

cd pages/Projets/GestionDuPersonnel-MVC-main/S2-MVC
php -S localhost:8765

Ouvrez ensuite http://localhost:8765 dans le navigateur. Adaptez le port si celui-ci est déjà utilisé.
