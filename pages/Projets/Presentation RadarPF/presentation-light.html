Le cœur du système

Propulsé par Express.js, le serveur backend gère les requêtes avec une vélocité remarquable. Il sert d'intermédiaire fiable entre la base de données brute et l'interface utilisateur web.

  • API RESTful légère (/api/radars)
  • Routage optimisé
  • Gestion des erreurs asynchrone

Données structurées

Plus de 3200 points d'intérêt stockés dans une base SQLite locale (radars.db). Cette approche « fichier unique » garantit une portabilité exceptionnelle sans nécessiter de serveur SGBD lourd.

  • Requêtes SQL ultra-rapides
  • Script d'import automatisé (import-db.js)
  • Filtrage natif par type (ETF, ETD...)

Rendu spatial interactif

L'interface front-end exploite Leaflet et les tuiles OpenStreetMap. Le résultat ? Une carte fluide, zoomable, intégrant nativement la géolocalisation de l'utilisateur.

  • Marqueurs colorés par catégorie
  • Pop-ups informatives au clic
  • Positionnement GPS temps réel
index.js (Express Route)

// Route principale pour récupérer les radars
app.get('/api/radars', (req, res) => {
    const type = req.query.type;
    let sql = 'SELECT * FROM radar';
    let params = [];
    
    if (type && type !== 'Tous') {
        sql += ' WHERE type = ?';
        params.push(type);
    }
    
    db.all(sql, params, (err, rows) => {
        if (err) return res.status(500).json({error: err.message});
        res.json(rows); // +3000 radars en ~15ms
    });
});

const map = L.map('map').setView([46.603354, 1.888334], 6);

L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
    attribution: '© OpenStreetMap contributors'
}).addTo(map);

function renderMarkers(radars) {
    radars.forEach(radar => {
        const color = getColorByType(radar.type);
        const marker = createCustomMarker(radar.latitude, radar.longitude, color);
        marker.bindPopup(`Type: ${radar.type}
VMA: ${radar.vma} km/h`).addTo(map);
    });
}
