Le Cerveau : Express.js

Le serveur web reçoit les coordonnées via une requête GET /weather/:lat/:long, s'assure de l'intégrité des données reçues et orchestre asynchronement la suite du pipeline.

  • Validation des types ISO
  • Gestion fluide des erreurs via try/catch
index.js
app.get('/weather/:lat/:long', async (req, res) => {
    const { lat, long } = req.params;

    if (isNaN(lat) || isNaN(long)) {
        return res.status(400).json({
            error: 'Coordonnées invalides'
        });
    }

    try {
        const apiKey = process.env.WEATHER_API_KEY;
        const apiUrl = `https://api.weatherapi.com/v1/current.json?key=${apiKey}&q=${lat},${long}&aqi=yes`;
        const response = await fetch(apiUrl);
        const json = await response.json();
        res.json(json);
    } catch (err) {
        res.status(500).json({ error: err.message });
    }
});

La Mémoire : SQLite

Le fichier weatherAppapi.db vit à côté du serveur Node. Chaque enregistrement d’humeur crée une ligne traçable : qui (session implicite), où (lat/long), quoi (mood), quand (timestamp ISO). Idéal pour prototyper sans base cloud.

Ce que stocke la base

Colonne Rôle
idClé primaire auto-incrémentée
latitude / longitudePosition arrondie associée au mood
moodLibellé ou code d’humeur
created_atHorodatage de l’enregistrement
  • Fichier unique — zéro configuration DBA
  • Transactions SQLite pour inserts sûrs
  • Réponses JSON légères pour le front
schema.sql
CREATE TABLE IF NOT EXISTS locations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  latitude REAL NOT NULL,
  longitude REAL NOT NULL,
  mood TEXT NOT NULL,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP
);

Prochaine étape

Prêt à explorer la météo ?

Installez les dépendances, configurez votre clé API météo dans .env, puis lancez le serveur Express. Ouvrez l’app sur le port affiché dans le terminal (souvent 3000).

  1. npm install à la racine du projet
  2. WEATHER_API_KEY renseignée pour WeatherAPI
  3. npm start — puis ouvrez l’URL locale dans le navigateur
