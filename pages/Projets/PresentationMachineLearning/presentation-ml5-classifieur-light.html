00 Contexte du projet

MachineLearningPF est un classificateur d'images qui fait tourner un réseau de neurones directement dans le navigateur, sans serveur ni API cloud. L'objectif : démontrer que l'apprentissage automatique est accessible en JavaScript, avec une expérience fluide et des résultats en français.

Pourquoi MobileNet ?

MobileNet est un CNN (réseau de neurones convolutif) léger, conçu par Google pour l'edge computing. Pré-entraîné sur ImageNet, il reconnaît plus de 1000 classes d'objets courants. Son petit gabarit permet une inférence en quelques millisecondes dans le navigateur via WebGL.

Pourquoi ML5.js ?

ML5.js est une bibliothèque « créative » qui encapsule TensorFlow.js. Elle simplifie l'accès aux modèles pré-entraînés : une seule ligne ml5.imageClassifier('MobileNet') suffit à charger le modèle et le rendre prêt à l'emploi.

Sources d'entrée multiples

Le projet accepte trois types d'entrée : glisser-déposer (fichier local), webcam (capture instantanée via Canvas) et images de test (galerie intégrée). Chaque image passe par le même pipeline : chargement → affichage → classification Top 5.

01 Du pixel à la prédiction

Entrée

L'image est chargée via FileReader, getUserMedia (webcam) ou URL directe. MobileNet attend un format 224×224 pixels normalisé. ML5 et TensorFlow.js gèrent cette prétraitement automatiquement.

Inférence

model.classify(img, 5) lance le forward pass. Les couches convolutives, ReLU et pooling s'exécutent sur le GPU via WebGL. Le résultat : un tableau de 5 prédictions avec label et confidence.

Sortie

Chaque prédiction est traduite via le dictionnaire TRADUCTIONS (200+ entrées ImageNet en français). L'affichage inclut barres de confiance, rang et temps d'inférence en millisecondes.

02 Extraits de code

Voici les éléments essentiels qui structurent le projet — initialisation du modèle, classification, traduction des labels et gestion des entrées multiples.

Initialisation MobileNet & Classification

Le modèle est chargé au démarrage. Dès qu'une image est prête, classify(img, 5) retourne les 5 meilleures prédictions. On mesure le temps avec performance.now().

js/demo-ml5-classifieur.js 
// Chargement du modèle au démarrage
ml5.imageClassifier('MobileNet')
    .then((model) => {
        state.model = model;
        state.modelReady = true;
    });

// Classification — Top 5 prédictions
function classify() {
    const start = performance.now();
    state.model.classify(DOM.imagePreview, 5)
        .then((results) => {
            const elapsed = Math.round(performance.now() - start);
            renderResults(results, elapsed);
        });
}

Traduction française des labels

ImageNet fournit des labels en anglais. Un dictionnaire TRADUCTIONS permet d'afficher « banane », « chat tigré » ou « retriever doré » à la place de « banana », « tabby cat » et « golden retriever ».

js/demo-ml5-classifieur.js 
const TRADUCTIONS = {
    'banana': 'banane', 'strawberry': 'fraise',
    'tabby cat': 'chat tigré', 'golden retriever': 'retriever doré',
    // ... 200+ entrées ImageNet
};

function traduire(label) {
    const key = label.toLowerCase().trim().split(',')[0].trim();
    if (TRADUCTIONS[key]) return TRADUCTIONS[key];
    return label.charAt(0).toUpperCase() + label.slice(1);
}

Glisser-déposer & chargement d'image

Les événements drop et change (file input) déclenchent un FileReader qui convertit le fichier en Data URL. L'image est assignée à img.src ; l'événement load déclenche la classification.

js/demo-ml5-classifieur.js 
DOM.dropZone.addEventListener('drop', (e) => {
    const file = e.dataTransfer.files[0];
    if (file && file.type.startsWith('image/')) {
        const reader = new FileReader();
        reader.onload = (ev) => loadImage(ev.target.result, 'file');
        reader.readAsDataURL(file);
    }
});

DOM.imagePreview.addEventListener('load', () => {
    state.imageLoaded = true;
    tryClassify(); // Lance la classification quand l'image est prête
});

Capture webcam via Canvas

La webcam utilise getUserMedia. Au clic sur « Capturer », on dessine la frame vidéo courante dans un Canvas, puis on exporte en Data URL avec toDataURL('image/jpeg'). Le flux passe par le même pipeline que les autres sources.

js/demo-ml5-classifieur.js 
// Ouverture webcam
const stream = await navigator.mediaDevices.getUserMedia({ video: true });
DOM.webcamVideo.srcObject = stream;

// Capture et classification
function captureFromWebcam() {
    const canvas = document.createElement('canvas');
    canvas.width = DOM.webcamVideo.videoWidth;
    canvas.height = DOM.webcamVideo.videoHeight;
    canvas.getContext('2d').drawImage(DOM.webcamVideo, 0, 0);
    loadImage(canvas.toDataURL('image/jpeg'), 'webcam');
}

État centralisé & architecture IIFE

L'application utilise un objet state unique et une IIFE (Immediately Invoked Function Expression) pour encapsuler tout le code. Pas de fuite de variables globales, pas de framework — uniquement du JavaScript vanilla.

js/demo-ml5-classifieur.js 
(function () {
    'use strict';

    const state = {
        model: null,
        modelReady: false,
        imageLoaded: false,
        currentSource: null,  // 'file' | 'webcam' | 'sample'
        webcamStream: null,
    };

    const DOM = { dropZone, fileInput, imagePreview, ... };

    function initModel() { ... }
    function classify() { ... }
    function renderResults(results, elapsed) { ... }

    init();
})();
