Perspective

Déjà « demain » dans le navigateur

Ce projet assume une conviction : la géodata spatiale n’a pas besoin d’un stack obscur pour être spectaculaire — un flux HTTP propre, du JSON et une carte suffisent pour donner l’impression d’un centre de contrôle orbital.

Ouverture sans clé API

Endpoint documenté, pas d’authentification payante : la démo vit dans le même monde que les données publiques spatial awareness.

Planète dans le DOM

Leaflet impose une lecture intuitive : la Station devient un pixel qui traverse continents et océans en temps quasi réel.

Rythme / silence

Polling à intervalle fixe plutôt que torrent WebSocket : suffisant pour l’ISS, élégant pour la batterie et le réseau.

Schéma d’orbite stylisé · CSS pur

Données

Du satellite au pixel

Cliquez sur une étape pour détailler le flux appliqué dans index.js.

  1. fetch(API_URL) interroge l’endpoint REST ; les erreurs réseau ou HTTP sont capturées pour afficher un message utilisateur sans bloquer la page.

  2. La réponse est parsée en objet JavaScript : latitude, longitude, altitude, vitesse et horodatage alimentent le DOM et la carte.

  3. Au premier jeu de coordonnées valides, la carte est initialisée ; ensuite le marqueur est déplacé et la vue suit avec panTo animé.

  4. setInterval(fetchISS, POLL_MS) relance la récupération toutes les 4 secondes pour garder un suivi quasi temps réel sans recharger la page.

Contexte

Vue d’ensemble

Le dépôt pages/Projets/iss.js-main regroupe une page index.html, la logique index.js et les dépendances Leaflet chargées en CDN. L’objectif pédagogique est clair : consommer une API tierce documentée, manipuler des données géospatiales et les rendre sur une carte web standard.

L’ancienne présentation embarquée dans le dossier (iss.js-main/presentation.html) a été remplacée par cette page portfolio pour garder une navigation et un ton alignés sur le reste du site.

Technique

Découper le projet

Trois angles pour retrouver vite ce qui vous intéresse dans les sources.

  • URL : https://api.wheretheiss.at/v1/satellites/25544/
  • Méthode asynchrone avec async/await et contrôle response.ok
  • Validation minimale des coordonnées avant init carte
  • L.map('issMap') avec worldCopyJump pour le méridien
  • Tuiles OSM via L.tileLayer et attribution respectée
  • L.divIcon pour un marqueur léger basé sur emoji
  • index.html — structure, stats, conteneur carte
  • index.js — logique fetch, rendu et timer
  • Leaflet 1.9 chargé en CDN (CSS + JS)

Exécution

Lancer la démo en local

Servez le dossier iss.js-main en HTTP pour éviter les blocages sur les tuiles et garantir un comportement fetch cohérent.

cd pages/Projets/iss.js-main
python3 -m http.server 8080

Puis ouvrez http://localhost:8080 (adaptez le port). La même API est utilisée par le bloc « live » en haut de cette page lorsque vous êtes en ligne.
